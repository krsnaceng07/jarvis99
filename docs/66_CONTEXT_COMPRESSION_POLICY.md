# 66_CONTEXT_COMPRESSION_POLICY.md

## Purpose
This document defines the Context Compression Policy for JARVIS OS. It establishes the guidelines for sliding windows, semantic pruning, vector summarization, and key-value retrieval.

## Scope
Applies to the Context Engine, vector database retrievers, and token counting loops inside the Brain Core.

## Context Compression Policies
1. **The 50% Context Gate:** When active context token size exceeds 50% of the model's maximum limit, the Context Engine must initiate compression.
2. **Context Compression Methods:**
   - **Semantic Pruning:** Remove non-essential logs, comments, styling elements, and duplicate outputs.
   - **Sliding Window:** Keep only the last 3 turns of raw conversation history. Earlier turns are replaced by a single paragraph summary generated by a local model.
   - **Vector Summarization:** Large target code files are summarized, mapping class interfaces, imports, and function signatures. The raw file content is only loaded when editing is requested.
3. **Hierarchy of Retention:** During compression, the system must retain Project Constitution blocks, active task checklist details, and error tracebacks.

## Responsibilities
- **Context Engine:** Computes token lengths, executes summarization hooks, and prunes low-priority variables.
- **Model Router:** Rejects payloads that exceed target model context windows.

## Dependencies
- Must strictly adhere to the [00_PROJECT_CONSTITUTION.md](file:///e:/jarvis/docs/00_PROJECT_CONSTITUTION.md) (specifically Rule 2, Rule 6, and Rule 10).

## Interfaces
- Local API helper: `ContextEngine.compress(payload: ContextPayload)`.

## Examples
- **Correct Compression:** Agent session reaches 60,000 tokens. Context Engine compresses conversation logs into a 500-token summary and replaces imports with skeleton interfaces, reducing size to 15,000 tokens.
- **Incorrect Compression:** Agent session reaches limit. Context Engine truncates the Project Constitution or active task details to save space. (Violates Hierarchy of Retention rules).

## Failure Cases
- **Loss of Crucial Context:** Important variables are pruned during compression, leading to reasoning bugs. *Mitigation:* The agent runs validation checks on compressed context payloads. If vital task info is missing, it requests retrieval from the long-term database.

## Security Considerations
- Context compression algorithms must run locally (e.g. using local models or regex parsers) to prevent data leak loops.

## Future Extension
- Enhancing compression algorithms (e.g. implementing LLMLingua) is managed through ADR logs.
