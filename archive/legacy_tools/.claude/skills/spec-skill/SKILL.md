# Spec Skill

## Purpose
Before building anything, this skill should turn my rough request into a clear implementation plan.

## Instructions
1. Understand the goal.
2. Ask only necessary clarification questions.
3. Identify files, components, APIs, database changes, and dependencies involved.
4. Break the task into small build steps.
5. Define success criteria.
6. List possible edge cases.
7. Create a review checklist that can be used later by the Review Skill.

## Output Format
- Goal
- Scope
- Files likely involved
- Step-by-step build plan
- Edge cases
- Success criteria
- Review checklist
