# Review Skill

## Purpose
This skill should review the work produced by the Build Skill against the original Spec Skill plan.

## Instructions
1. Compare the final output with the original goal.
2. Check every success criterion.
3. Check every edge case.
4. Inspect changed files.
5. Identify missing logic, bugs, weak code, or incomplete work.
6. Suggest exact fixes.
7. Send the fixes back into the Build Skill if required.

## Review Style
Be strict. Do not approve weak or incomplete work.

## Output Format
- Pass/fail summary
- Requirements met
- Requirements missed
- Bugs or risks found
- Edge cases missed
- Fixes needed
- Final verdict: approved / needs another build loop
